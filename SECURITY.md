# Security Policy

WinMinimal modifies Windows configuration.

Security-related issues should be handled carefully and responsibly.

---

# Reporting a Vulnerability

If you discover a security issue, please do not publish it immediately.

Open a private discussion or contact the maintainer so the issue can be evaluated before public disclosure.

When reporting a vulnerability, include:

- affected WinMinimal version;
- affected Windows version;
- steps to reproduce;
- expected behavior;
- actual behavior;
- relevant logs, if available.

---

# Scope

Security issues may include:

- unsafe default settings;
- unintended privilege requirements;
- incorrect registry changes;
- unexpected weakening of Windows security features;
- behavior that contradicts the documented intent of a module.

---

# Project Approach

WinMinimal is designed to be conservative.

Security-sensitive changes should be:

- explicit;
- configurable;
- documented;
- logged;
- reversible whenever possible.

---

# Disclaimer

WinMinimal is provided without warranty.

Users are responsible for reviewing the configuration before execution and validating that it matches their environment and security requirements.
