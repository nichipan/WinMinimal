\# Contributing



Thank you for your interest in WinMinimal!



The project is still in its early stages, but contributions, ideas and constructive feedback are welcome.



\---



\## Project Principles



WinMinimal follows a few simple engineering principles:



\- Small modules

\- One responsibility per module

\- Explicit configuration

\- Simple and readable code

\- Deterministic behavior

\- Repeatable execution

\- Clear documentation



\---



\## Coding Style



\- Use PowerShell approved verbs whenever possible.

\- Keep functions short.

\- Prefer explicit code over clever code.

\- Write comments in English.

\- Keep configuration separate from implementation.



\---



\## Pull Requests



Before submitting a Pull Request:



\- Test your changes.

\- Update the documentation if needed.

\- Keep commits focused on a single logical change.
